# Event Page Builder Prebuilt Components - Comprehensive Prompt Guide

## 1. EVENT HERO SECTIONS

**Component Purpose**: Capture attention and communicate event essence with key details at first glance

**Required Variations**:
- Full-screen hero with event date countdown
- Split hero (event visual + key info card)
- Video background hero with registration CTA
- Animated gradient hero with floating event badges
- Parallax hero with layered event elements
- Minimal hero with large typography focus

**Design Specifications**:
- Height: 100vh, 85vh, or 700-900px options
- Date Display: Large, prominent date format (multiple styles: "March 15, 2025", "15 MAR 2025", countdown timer)
- Typography: Event name (56-84px bold), tagline (24-28px), location/time (18-20px)
- Countdown Timer: Days, hours, minutes, seconds in separate boxes (48-72px numbers)
- Quick Info Bar: Floating card with date, location, price, capacity at a glance
- CTA Buttons: "Register Now", "Get Tickets", "Learn More" - multiple button slots
- Media Support: High-quality event imagery, venue shots, past event videos
- Overlay Options: Dark gradient (50-70% opacity) for text readability
- Badge Elements: "Early Bird", "Limited Spots", "Virtual Available", "Sold Out"
- Responsive: Stack info vertically on mobile, maintain countdown visibility
- Animation: Fade-in event details, pulsing register button, countdown ticking
- Accessibility: High contrast (4.5:1 minimum), alt text for images, timer with ARIA labels

## 2. EVENT DETAILS/INFO SECTIONS

**Component Purpose**: Present comprehensive event information in scannable, organized format

**Required Variations**:
- Icon-based info grid (date, time, location, price, capacity, category)
- Timeline format with visual progression
- Tabbed information (overview, schedule, speakers, venue)
- Expandable accordion with detailed sections
- Two-column layout (essentials + description)
- Interactive map with event location

**Design Specifications**:
- Icon Size: 56-72px, custom icons for each info type
- Info Cards: 240-320px width, 180-240px height, with shadow/border
- Typography: Label (14-16px uppercase), value (20-24px bold), description (16-18px)
- Color Coding: Different accent colors for different info types (date=blue, location=red, price=green)
- Map Integration: Embedded Google Maps or Mapbox, 400-600px height
- Visual Hierarchy: Most important info (date/time) larger and more prominent
- Venue Details: Address, parking info, public transport, accessibility features
- Capacity Indicator: Progress bar showing seats remaining (e.g., "78/100 spots filled")
- Price Display: Multiple ticket tiers if applicable, strikethrough for early bird savings
- Category Tags: Event type badges (Conference, Workshop, Festival, Networking, etc.)
- What to Bring: Checklist format with checkbox icons
- Responsive: 3 columns → 2 columns → 1 column, map full-width on mobile

## 3. SCHEDULE/AGENDA SECTIONS

**Component Purpose**: Display event timeline and activities in clear, navigable format

**Required Variations**:
- Vertical timeline with time markers
- Multi-day tabbed schedule
- Grid view (time slots × tracks/rooms)
- List view with collapsible details
- Calendar-style day view
- Now-playing live schedule (during event)

**Design Specifications**:
- Time Display: 24px bold, 12/24-hour format options, timezone indicator
- Time Markers: Vertical line with circular nodes (16-24px)
- Session Cards: Title (20-24px), speaker (16px), duration (14px), location (14px)
- Track/Room Colors: Color-coded sessions by track (up to 8 color variations)
- Card Spacing: 16-24px between sessions, 40-60px between day sections
- Hover State: Expand card to show full description, speaker bio preview
- Interactive Elements: Click to add to personal schedule, export to calendar
- Break Indicators: Different styling for breaks, lunch, networking (lighter background)
- Current Time Indicator: Moving line/highlight showing current position (for live events)
- Duration Bars: Visual bars showing session length proportionally
- Filter Options: By track, by speaker, by session type, by time
- Search Function: Real-time search through session titles and speakers
- Icons: Coffee icon for breaks, presentation icon for talks, discussion icon for panels
- Responsive: Grid becomes list on mobile, sticky time headers

## 7. FAQ SECTIONS (EVENT-SPECIFIC)

**Component Purpose**: Address common attendee questions and reduce support inquiries

**Required Variations**:
- Categorized accordion (Registration, Venue, Schedule, Refunds, etc.)
- Search-enabled FAQ with instant filtering
- Two-column layout for quick scanning
- Video FAQ answers for complex questions
- Live chat bot integrated with FAQ
- Expandable categories with nested questions

**Design Specifications**:
- Category Tabs: 48px height, clear icons, active state highlighting
- Question Display: 18-20px medium weight, chevron/plus icon (24px)
- Answer Text: 16px, line-height 1.7, max-width 800px for readability
- Spacing: 16-20px between questions, 32-40px between categories
- Search Bar: Sticky at top, 56px height, magnifying glass icon, "Search FAQs..."
- Animation: 300ms smooth expand/collapse with fade-in content
- Common Questions: Pre-expanded or highlighted (e.g., "What's included in ticket?")
- Contact Link: "Still have questions? Contact us" at section bottom
- Icons: Question type icons (ticket, location, wifi, food, parking)
- Background: Alternating light gray/white rows or card-style with shadows
- Character Count: Questions 50-100 chars, answers 100-300 words
- Related Links: Link to relevant sections (e.g., FAQ about refunds links to refund policy)

## 10. COUNTDOWN/URGENCY SECTIONS

**Component Purpose**: Create urgency and drive immediate action through time-sensitive messaging

**Required Variations**:
- Large countdown timer with days/hours/mins/secs
- Progress bar showing registration deadline approaching
- Floating sticky countdown bar
- Early bird pricing expiration timer
- Spots remaining indicator with live updates
- Milestone countdowns (e.g., "100 days until event!")

**Design Specifications**:
- Timer Design: Flip-clock style or digital display, 48-96px numbers
- Box Styling: Separate boxes for D:H:M:S with labels below (12-14px)
- Color Urgency: Change colors as time decreases (blue → yellow → red)
- Animation: Flip animation on number change, pulsing effect when < 24 hours
- Progress Bar: 16-24px height, color-coded (green → red), percentage label
- Spots Counter: Circular progress or horizontal bar, "23/100 spots left"
- Update Frequency: Timer updates every second, spots update real-time
- Expired State: "Event started!" or "Registration closed" messaging
- Background: High-contrast background (dark with light text or vice versa)
- Size Options: Large (full-width section), medium (inline), small (header bar)
- Call-to-Action: Integrated CTA button below timer ("Register Before It's Too Late")
- Timezone Display: Show countdown in viewer's timezone with indicator
- Mobile Optimization: Compact timer design, maintain readability at small sizes
- Accessibility: Time remaining also in text format for screen readers

## 14. CALL-TO-ACTION (EVENT-SPECIFIC)

**Component Purpose**: Drive conversions at strategic points throughout the page

**Required Variations**:
- Sticky header CTA bar (always visible)
- Full-width banner CTA between sections
- Floating action button (FAB) bottom-right
- Exit-intent popup CTA
- Multi-option CTA (Register, Volunteer, Sponsor)
- Post-event CTA (Watch Recording, Join Next Year)

**Design Specifications**:
- Button Sizing: Primary 56-64px height, secondary 48px height
- Button Text: "Register Now", "Get Your Ticket", "Save My Spot", "Join Waitlist"
- Color: High-contrast brand color, passes WCAG AA (4.5:1 contrast)
- Hover State: Darken 10%, lift with shadow, or scale 1.05x
- Loading State: Spinner animation, disabled state, "Processing..." text
- Icon Addition: Arrow, ticket, calendar icons inside button (24px)
- Sticky Bar: 60-70px height, shadow below, register button + spots remaining
- FAB: 64-72px circle, pulse animation, collapses to icon on scroll
- Banner Design: Full-width, 300-400px height, left-text + right-CTA layout
- Urgency Elements: Countdown, spots remaining, "Early Bird ends soon"
- Multi-CTA Layout: 3 options side-by-side, clear visual hierarchy
- Mobile Sticky: Fixed bottom bar with CTA, doesn't block content
- Accessibility: High contrast, min 44x44px touch target, focus indicators
- Analytics: Track click-through rates by CTA placement

## UNIVERSAL EVENT PAGE DESIGN PRINCIPLES

**Color Psychology for Events:**
- Corporate Events: Blues, grays, professional palette
- Tech Conferences: Vibrant blues, purples, electric accents
- Music Festivals: Bold, saturated colors, gradients
- Wellness Events: Greens, earth tones, calming pastels
- Fundraisers: Warm colors, reds, oranges for urgency
- Sports Events: Team colors, high energy reds/yellows

**Typography Hierarchy for Events:**
- Event Name: 56-84px, bold, eye-catching display font
- Section Headers: 36-48px, medium/bold weight
- Subheadings: 24-32px, regular/medium weight
- Body Text: 16-18px, line-height 1.6-1.8
- Details/Metadata: 14-16px, can be lighter weight
- Mobile Scale: Reduce by 20-30% for mobile

**Spacing System (8px base):**
- Tight: 8-16px (within components)
- Standard: 24-32px (between related elements)
- Comfortable: 48-64px (between sections)
- Spacious: 80-120px (between major sections)

**Event-Specific Breakpoints:**
- Mobile Portrait: < 480px
- Mobile Landscape: 480-767px
- Tablet: 768-1024px
- Desktop: 1025-1440px
- Large Desktop: > 1440px

**Animation Guidelines:**
- Page Load: Stagger animations, don't overwhelm
- Scroll Triggers: Fade-in sections at 20% viewport
- Hover States: 200-300ms transitions
- Countdown Timer: Smooth number transitions
- CTA Pulses: 2-second loop, subtle scale

**Accessibility Musts:**
- WCAG 2.1 AA minimum compliance
- 4.5:1 contrast for body text
- 3:1 contrast for large text (18px+)
- Focus indicators on all interactive elements
- Skip navigation links
- Proper heading hierarchy (H1→H2→H3)
- Alt text for all images
- Video captions and transcripts
- Keyboard-navigable components
- Screen reader tested
- Touch targets minimum 44x44px

**Performance Optimization:**
- Hero images: WebP format, < 500KB
- Lazy load images
